## 已确认的 NT 公式

来源：芯片 C 代码 + 芯片回读数据交叉验证

```c
// Flash 格式噪声阈值 (readbuf[i] = value_in_MT - 10)
teliconum = -gain_diff;   // Telecoil 输入时 = 45
mic1Cal = floor(sum(mic1_calibration_value[SNR_FreqSpacing[i] ... +cnt-1]) * 10 / cnt) / 10;

NT*10 = round(f32(5.307) * f32(readbuf[i] + 10 + 130 + teliconum - mic1Cal) - f32(371.2));
// 等效: NT*10 = round(5.307 * (185 - mic1Cal) - 371.2)   (readbuf=0, teliconum=45 时)
```

关键参数：
- `readbuf[i]` = Flash 格式噪声阈值 = `value_in_MT - 10` = 0（value_in_MT = 10）
- `teliconum` = 45（`telecoil_gain_diff = -450` → 0.1dB 单位 → `-(-45) = 45`）
- `mic1Cal` = 当前 ENR 频段区间内的校准值平均（floor 取整）
- 浮点精度：float32，结果 round 到 int12

## 核心问题

**公式和浮点精度已确认正确**，但 `mic1Cal` 的取值方式与 ENR 频段划分命令定义的 band 区间不一致。

芯片代码中的 `SNR_Frequency_Spacing[]` 数组决定了每个 ENR 通道在 NT 计算中使用哪个校准 band 区间。但实际芯片使用的有效 mic1Cal 值对应的 band 索引，与 ENR 命令中的频段区间没有明显的对齐关系。

具体例子：
- **ch2** (375 Hz)：band 范围 [2,3] = [151,152]，平均 = 151。但芯片需要 150，在 mic1_band 中对应索引 0, 1, 7, 17, 19 — 全部远离 ch2 的频段
- **ch6** (2375 Hz)：band 范围 [10,11] = [147,157]，平均 = 152。芯片需要 153，而 mic1_band 中完全没有值为 153 的 band（意味着必须通过多 band 平均得到）
- **ch13** (5875 Hz)：同样找不到精确匹配 144 的 band
