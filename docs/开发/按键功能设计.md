# BS300 按键功能设计文档

> 2026-07-13 | `peripheral_server_sleep` 项目
> 硬件：按键复用在 DIO12（与恢复模式共用引脚），低电平有效（按下导通 GND），内部弱上拉
> 恢复模式仅在开机初始化检查，与按键正常运行不冲突

## 1. 按键行为定义

| 操作 | 判定条件 | 行为 |
|------|---------|------|
| 短按 | 按住 < 1.5s | 音量 +1，0→1→...→9→0 循环 |
| 长按 | 按住 ≥ 1.5s | 切换模式，0→1→2→0 循环，跳过 Program 3（音频模式） |

## 2. 最终实现：文件结构

```
peripheral_server_sleep/
├── app.c                  # 全部按键逻辑内联于此（140 行：回调 + 状态机 + 动作分发）
├── code/app_process.c     # Continue_Application() 中添加 DIO12 重配（唤醒后恢复）
└── include/app.h          # 无需修改（DIO pin 号直接硬编码，无新增宏）
```

**无独立按键模块**。经过多轮 ISR / 独立文件 / 内联方案的对比测试，最终采用**纯内联轮询**方案，全部逻辑集中在 `app.c` 两处：`main()` 中的 DIO12 初始化 + `Main_Loop()` 中的 70 行状态机。

## 3. 架构设计

### 3.1 实际分层

```
┌─────────────────────────────────────────────────────┐
│  Main_Loop (app.c)                                  │
│  ├─ 5 次采样多数投票滤波                              │
│  ├─ 边沿检测 + hold 计数（~1ms/tick）                 │
│  ├─ 短按/长按判定 → pending_action                   │
│  ├─ bs300_set_volume_async() / bs300_switch_program_async() │
│  └─ 回调: on_btn_*_done() → lp_en = true             │
├─────────────────────────────────────────────────────┤
│  DIO12 初始化（两处必须）                               │
│  ├─ main()            → 首次启动时配置                 │
│  └─ Continue_Application() → 每次休眠唤醒后重配        │
├─────────────────────────────────────────────────────┤
│  硬件层                                              │
│  ├─ DIO12: GPIO_IN_0 + 弱上拉 + LPF_DISABLE           │
│  └─ 唤醒源: DIO12 FALLING edge（已配，无需改动）        │
└─────────────────────────────────────────────────────┘
```

**关键设计决策**：
- **不用 ISR**：RTE 不配 DIO12 中断（`RTE_GPIO_DIO12_INT_EN=0`），避免伪中断、NVIC 休眠丢中断等问题
- **不用 BLE 定时器**：hold 计时用主循环 `Sys_Delay_ProgramROM(~1ms)` 阻塞计数，不受 BLE 调度影响
- **DIO12 必须双入口配置**：首次启动在 `main()`，每次唤醒在 `Continue_Application()`，缺一不可

### 3.2 状态机

```
                         ┌──────────────┐
                 上电    │   RELEASED   │
              ─────────▶│ btn_now = 0  │
                        │ btn_prev = 0 │
                        └──────┬───────┘
                               │ DIO = 0（5 次采样多数为 0）
                               ▼
                        ┌──────────────┐
                        │   PRESSED    │
                        │ lp_en = false│
                        │ hold_ticks=0 │
                        └──────┬───────┘
                               │ DIO 持续 = 0
                               ▼
                        ┌──────────────┐
                        │    HELD      │
                        │ hold_ticks++ │─── 每 tick ~1ms ──▶ ticks ≥ 1500
                        │ lp_en = false│                     │
                        └──────┬───────┘                     ▼
                               │                    ┌──────────────┐
                               │ DIO = 1（释放）     │  LONG PRESS  │
                               ▼                    │ (提前触发)    │
                        ┌──────────────┐            │ pending=LONG │
                        │ ticks < 1500 │            └──────┬───────┘
                        │ → pending    │                  │ DIO = 1（释放）
                        │   = SHORT    │                  ▼
                        └──────────────┘         ┌──────────────┐
                               │                 │ I2C done?    │
                               ▼                 │ → lp_en=true │
                        ┌──────────────┐         └──────────────┘
                        │ 动作分发      │
                        │ !busy? → I2C │
                        │ done → lp_en │
                        └──────────────┘
```

### 3.3 长按阈值

```
HOLD_THRESHOLD = 1500 ticks × ~1ms = 1500ms ≈ 1.5s
```

按住期间每轮主循环执行 `Sys_Delay_ProgramROM(SystemCoreClock / 1000)`，约 1ms/tick。长按在 tick ≥ 1500 时**提前触发**（不等释放）。抬起时检查 `long_fired`，已触发的跳过短按判定。

## 4. 关键实现细节

### 4.1 DIO12 必须双入口初始化（最重要）

**现象**：按键代码在 DEBUG 打印模式下完全正常（DIO12 读到高电平），一关打印就不进休眠。

**根因**：休眠唤醒走 `Continue_Application()`，该函数重配了 LED、DIO4、DIO5，但未配置 DIO12。唤醒后 DIO12 回到默认浮空态，`DIO_DATA->ALIAS[2]` 读到 0 → 按键状态机判为"按下"边沿 → `low_power_enable = false` → 不再进休眠。

DEBUG 模式不走 sleep，`Continue_Application()` 永不被调用，DIO12 保持 `main()` 中配好的初始态，所以打印正常。

**解决**：两处都配 DIO12：

```c
// app.c → main() — 首次启动
Sys_DIO_Config(12, DIO_MODE_GPIO_IN_0 | DIO_WEAK_PULL_UP | DIO_LPF_DISABLE);

// app_process.c → Continue_Application() — 每次唤醒
Sys_DIO_Config(12, DIO_MODE_GPIO_IN_0 | DIO_WEAK_PULL_UP | DIO_LPF_DISABLE);
```

注意必须用 `DIO_MODE_GPIO_IN_0`（0x11），不能用 `DIO_MODE_INPUT`（0x01）。后者上拉不生效，DIO12 读到 0。

### 4.2 软件消抖：5 次采样多数投票

```c
uint8_t cnt_low = 0;
for (i = 0; i < 5; i++) {
    if (DIO_DATA->ALIAS[12] == 0) cnt_low++;
}
btn_now = (cnt_low >= 3) ? 1 : 0;
```

每轮主循环在同一位置连续读 5 次，≥3 次读到低电平才判定为按下。在同一次 poll 内完成，不跨循环迭代，避免休眠间隔导致的时序不确定性。

### 4.3 低功耗控制

```
按下边沿 → lp_en = false
按住中   → lp_en = false（每轮覆写，防止 I2C 回调恢复后被意外休眠）
短按抬起 → 不发 lp_en（等 I2C 回调恢复）
长按抬起 + I2C 已完 → lp_en = true
长按抬起 + I2C 未完成 → 不发 lp_en（等回调）
I2C 回调 → lp_en = true（如果按钮仍按住，下轮 HELD 分支会重新设 false）
```

`DEBUG_UART_ENABLE` 开启时 `#ifndef DEBUG_UART_ENABLE` 禁掉整段 sleep，不影响按键逻辑验证。

### 4.4 按键动作处理

```c
if (pending_action != BTN_NONE && !bs300_sync_is_busy()) {
    if (pending_action == BTN_LONG) {
        prog = bs300_get_active_prog();
        next = (prog + 1) % 3;  // skip Program 3
        bs300_switch_program_async(next, on_btn_switch_done);
    } else {
        vol = (app_env.volume + 1) % 10;
        app_env.volume = vol;
        bs300_set_volume_async(vol, on_btn_volume_done);
    }
    pending_action = BTN_NONE;
}
```

- `!bs300_sync_is_busy()` 守卫
- BLE 路径用 `on_bs300_*_done`（不恢复 lp_en），按键路径用 `on_btn_*_done`（恢复 lp_en）

```c
static void on_btn_switch_done(void) {
    cs_env.tx_value_changed = 1;
    bs300_async_done_callback();
    low_power_clk_param.low_power_enable = true;
}
```

## 5. 时序参考

```
短按 (< 1.5s):
  DIO↓ → edge → lp_en=false, hold=0
  [按住 ~200ms]
  DIO↑ → edge → pending=SHORT
  next loop → !busy → I2C ~10ms → callback → lp_en=true → sleep

长按 (≥ 1.5s):
  DIO↓ → edge → lp_en=false, hold=0
  [按住, 每 ~1ms: hold++]
  hold≥1500 → long_fired=1, pending=LONG
  next loop → !busy → I2C ~124ms → callback → lp_en=true
  [如果还按住: next loop HELD 分支 → lp_en=false]
  DIO↑ → edge → I2C done? → lp_en=true → sleep
```

## 6. 开发过程踩坑记录

### 6.1 独立按键模块方案被废弃

最初按设计创建了 `button.h` + `button.c` 独立模块，后因以下问题废弃：

1. **`stdbool.h` 依赖**：`button.h` 引入 `<stdbool.h>`，虽然 ARM GCC 支持，但增加了模块间依赖
2. **`DIO_MODE_INPUT` 上拉不生效**：必须用 `DIO_MODE_GPIO_IN_0`，独立模块调试时未能定位
3. **模块分离增加调试难度**：问题定位需要跨文件追踪

**最终方案**：内联到 `app.c`，零额外依赖，所有逻辑在 70 行内。

### 6.2 电平触发 → 边沿触发

第一版用每轮 `if (pressed) lp_en=false else lp_en=true`，导致开机时无条件覆写 `lp_en`，干扰了测量周期的 `lp_en` 管理。

改为边沿触发后，稳态不干预 `lp_en`，只在按下/抬起瞬间修改。

### 6.3 DIO12 配置入口缺失

见 4.1 节，这是耗时最长的 debug。

### 6.4 长按回调后被 HELD 分支覆写

长按 I2C 完成后回调设 `lp_en=true`，但下一轮 HELD 分支无条件设 `lp_en=false`。抬起边沿必须显式恢复 `lp_en=true`：

```c
else if (!btn_now && btn_prev) {
    if (!long_fired)
        pending_action = BTN_SHORT;
    else if (!bs300_sync_is_busy())
        low_power_clk_param.low_power_enable = true;  // ← 必须显式恢复
}
```

## 7. 调试方法

需要调试按键时，在 `app.h` 打开 `#define DEBUG_UART_ENABLE`，串口可看到：
- `[BTN] SHORT PRESS (ticks=xxx)` — 短按
- `[BTN] LONG PRESS (ticks=xxx)` — 长按

注意 DEBUG 模式下 sleep 被禁用，仅用于验证按键逻辑。
