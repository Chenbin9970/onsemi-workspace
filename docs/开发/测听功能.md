# 测听功能

> 2026-07-20 | 基于 peripheral_server_sleep

## 1. 协议概述

App 通过 BLE HDLC 协议控制 BS300 进入纯音测听模式，发送特定频率/分贝的纯音，完成后退出。

### 1.1 BLE 指令

| CMD ID | 名称 | 请求 | 响应 | 说明 |
|--------|------|------|------|------|
| 13 | SetPlayVoice | `[DevType][Spectrum][Decibel]` | `[Flag=0][status=1]` | 播放纯音 |
| 14 | SetStopVoice | `[DevType]` | `[Flag=0][status=1]` | 停止纯音 |
| 40 | SetAudiometryStatus | `[DevType][Fitting_Status]` | `[Flag=0][status=1]` | 进入/退出测听 |

**CMD 13 参数**：

| Spectrum | 频率 Hz | Spectrum | 频率 Hz |
|:--:|------|:--:|------|
| 0 | 250 | 9 | 4500 |
| 1 | 500 | 10 | 5000 |
| 2 | 1000 | 11 | 5500 |
| 3 | 1500 | 12 | 6000 |
| 4 | 2000 | 13 | 6500 |
| 5 | 2500 | 14 | 7000 |
| 6 | 3000 | 15 | 7500 |
| 7 | 3500 | 16 | 8000 |
| 8 | 4000 | | |

Decibel: 20-100 dB，步进 5 dB。

**CMD 40 Fitting_Status**：

| 值 | 含义 |
|:--:|------|
| 0 | 进入测听 |
| 1 | 退出测听 |

### 1.2 主动推送（Device → App）

| CMD ID | 名称 | 数据 | 说明 |
|--------|------|------|------|
| 6 | Receive device initial Status | `[DevType=1][Status]` | SYS_ID=1 |

| Status | 含义 | 触发时机 |
|:--:|------|------|
| 2 | 初始化完成 | 进入测听后 2s（DSP 稳定后） |
| 1 | 未初始化 | 退出测听后 2s |

## 2. DSP 流程

### 2.1 进入测听

```
CMD 40 status=0
  → hdlc_response (立即回复 ACK)
  → bs300_audiometry_enter():
      mute
      → 禁用 ENR (0x8000C2 全零)
      → 禁用 DDM2 (0x800022 全零)
      → 禁用 MM+ (0x800062 全零)
      → 禁用 DFBC (0x800052 全零)
      → 禁用 NoiseGen2 (0x800172 全零)
      → 禁用 ISS (0x8001B2 全零)
      → 禁用 WNR (0x8001C2 全零)
      → 禁用 AGCO (0x800382 全零)
      → 写入测试 WDRC 参数（固定值）:
          WDRC General (0x8000B2)
          WDRC Freq Spacing (0x8010B2)
          WDRC KP Threshold (0x8020B2)
          WDRC Ratio (0x8050B2)
          WDRC Bin Gain (0x8060B2)
      → active
  → state = TEST
  → ke_timer 2000ms
  → Push CMD=6 [01][02]（通知 App DSP 就绪）
```

### 2.2 播放纯音

```
CMD 13 (Spectrum, Decibel)
  → mute
  → bs300_itg_write(decibel, freq_hz, calib)
      → bs300_encode_itg() 编码 48B 数据
          itg_level   ← beep_frac24_table[decibel - mic1_band + 255]
          itg_freq    ← freq_hz / 250
          itg_select  ← 0x000001
      → bs300_advanced_write(0x8001E2)
  → active (DSP 开始输出纯音)
```

### 2.3 停止纯音

```
CMD 14
  → mute
  → bs300_itg_clear()
      → bs300_advanced_write(0x8001E2, 全零)
```

### 2.4 退出测听

```
CMD 40 status=1
  → hdlc_response (立即回复 ACK)
  → bs300_audiometry_exit():
      state = NONE
      mute
      → bs300_itg_clear()
      → bs300_sync_program(&s_dsp_state)  (全量 31 条 I2C 恢复)
      → active
  → ke_timer 2000ms
  → Push CMD=6 [01][01]（通知 App 已退出）
```

## 4. 状态管理

```c
typedef enum {
    BS300_AUDIOMETRY_NONE = 0,  // 正常模式
    BS300_AUDIOMETRY_TEST = 1,  // 测听模式（CMD 40 status=0）
} bs300_audiometry_state_t;
```

- `bs300_get_audiometry_state() != NONE` 时，按键处理应拦截切模式/调音量操作
- `bs300_audiometry_exit()` 内部自动清零状态

## 5. 按键拦截

测听/试听期间禁止按键打断，需在按键处理入口加入判断：

```c
if (bs300_get_audiometry_state() != BS300_AUDIOMETRY_NONE) return;
```

## 6. 调试打印

所有打印通过 `PRINTF(...)` 宏，受 `DEBUG_UART_ENABLE` 控制。关闭宏后自动展开为 `((void)0)`。

`ble_std.c` 已补充 PRINTF fallback 宏，防止关打印时链接报错。
