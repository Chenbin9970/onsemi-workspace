# 硬件配置

> 所有配置参数的定义位置、可选值、当前值。与 BLE 协议参数和调试打印无关的硬件层面配置。

---

## 一、射频发射功率 (`calibration.h`)

**`RF_TX_POWER_LEVEL_DBM`** — RF 输出功率

| 选项 | VCC 目标（MANU_CALIB） | 适用场景 |
|------|----------------------|----------|
| `0` | 1.10V | 低功耗、近距离 |
| `3` | 1.20V | 中等距离 |
| `6` | 1.20V + PA | 远距离（需启用 POWER_AMPLIFIER） |

当前值：**0 dBm**

---

## 二、校准策略 (`calibration.h`)

**`CALIB_RECORD`** — 电压校准方式

| 选项 | 说明 | 适用场景 |
|------|------|----------|
| `MANU_CALIB` | 从 NVR4 读取出厂校准值 | 量产芯片 |
| `SUPPLEMENTAL_CALIB` | 从 NVR3 读取补充校准值 | 已用 supplemental_calibrate 工具校准过 |
| `USER_CALIB` | 运行时自动计算并校准 | 开发板、未校准芯片 |

当前值：**MANU_CALIB**

## 三、系统时钟 (`app.h`)

### 3.1 主时钟

**`RFCLK_FREQ`** — 48M 晶振分频后的系统时钟

| 选项 | 典型用途 |
|------|---------|
| `8000000` (8 MHz) | 最低功耗 |
| `16000000` (16 MHz) | RM 2Mbps 协议最低要求 |
| `48000000` (48 MHz) | 最高性能 |

当前值：**8 MHz**

### 3.2 低功耗时钟源 (RTC)

**`RTC_CLK_SRC`** — Sleep 期间的 RTC 时钟

| 选项 | 精度 | 功耗 | 说明 |
|------|------|------|------|
| `RTC_CLK_SRC_XTAL32K` | 高 (ppm 级) | 较高 | 需外部 32K 晶振 |
| `RTC_CLK_SRC_RC_OSC` | 低 (±2%) | **最低** | 内部 RC 振荡器，AUDIOSINK 动态校准 |
| `RTC_CLK_SRC_DIO[0-3]` | 取决于外部源 | 最低 | 外部时钟输入 |

当前值：**RC_OSC**

### 3.3 RC 振荡器校准参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `RCCLK_FREQUENCY_SAMPLES` | 1000 | 初次测量样本数 |
| `LOW_POWER_CLK_INITIAL_MEASUREMENT` | 1000 | 初次测量周期数 |
| `LOW_POWER_CLK_DYNAMIC_MEASUREMENT` | 200 | 后续动态测量周期数 |
| `LOW_POWER_CLK_MEASUREMENT_INTERVAL_S` | 4 | 重新校准间隔（秒） |

### 3.4 XTAL32K 微调（仅 XTAL32K 模式）

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `XTAL32K_ITRIM_VALUE` | `0xF` | 电流 trim（0x0–0xF） |
| `XTAL32K_CLOAD_TRIM_VALUE` | `0x38` | 负载电容 trim |

## 七、GPIO 功能定义 (`app.h`)

| 引脚 | 功能 | 说明 |
|------|------|------|
| DIO 3 | **ADC 电池电量检测** | 1M+360k 分压输入，ADC ch0 采样 |
| DIO 4 | UART RX (调试) / 禁用 (低功耗) | DEBUG_UART_ENABLE 控制 |
| DIO 5 | UART TX (调试) / 禁用 (低功耗) | DEBUG_UART_ENABLE 控制 |
| DIO 6 | LED | 上电亮，进 sleep 前灭 |
| DIO 12 | Recovery / 按键 | 低电平时暂停启动，便于重新烧录 |

### 7.1 DIO3 ADC 电池检测（2026-07-21）

**硬件**：电池 4.4V → 1MΩ → DIO3 → 360kΩ → GND

| 参数 | 值 | 位置 |
|------|-----|------|
| `BAT_ADC_DIO` | 3 | `app.h` |
| `BAT_ADC_CHANNEL` | 0 | `app.h` |
| `BAT_ADC_MIN` | `0x0CC0` (DIO3=0.794V, 电池 3.0V) | `app.h`，需实测校准 |
| `BAT_ADC_MAX` | `0x12A0` (DIO3=1.165V, 电池 4.4V) | `app.h`，需实测校准 |

**ADC 配置**（4 处初始化均改用 DIO3）：
- `App_sleep_Initialize`：`ADC_NORMAL | ADC_PRESCALE_200`
- `App_RM_BLE_Initialize`：`ADC_NORMAL | ADC_PRESCALE_6400`
- `App_Initialize`：`ADC_NORMAL | ADC_PRESCALE_6400`
- `Continue_Application`：`ADC_NORMAL | ADC_PRESCALE_200`（每次 wakeup 重配）

**测量流程**：DIO3 → ADC ch0 → 16 次平均 → 0-100% → `app_env.batt_lvl` → BASS BLE 通知 + REMPRO CMD=4 查询

**数据上报**：
- BLE BASS：`Batt_LevelUpdateSend`（原 VBAT/2 路径不变，只是数据源改为 DIO3）
- REMPRO `GetBatteryInfo`（CMD=4）：返回 `app_env.batt_lvl`（之前硬编码 100%）

## 八、调试与低功耗切换

所有硬件行为由 `include/app.h` 中的 `DEBUG_UART_ENABLE` 宏控制：

| 硬件行为 | 宏启用 (调试) | 宏禁用 (低功耗) |
|----------|-------------|---------------|
| 电压校准 (NVR4) | 跳过 | 执行 |
| DIO4/5 | UART 开启 | 关闭 (no-pull) |
| Sleep 模式 | 跳过 | 进入 POWER_MODE_SLEEP |
| WFI | 跳过 | 执行 |
| printf 输出 | 有 | 无 |

详见 [调试打印.md](调试打印.md)。

## 九、远程麦 (RM) 音频硬件 (`APP_RM_ENABLE` 启用时)

| 参数 | 值 | 说明 |
|------|-----|------|
| 协议 | 私有 2.4GHz | 与 BLE 共存 (BBIF_COEX) |
| 数据速率 | 2 Mbps | 跳频 7 通道 |
| 编解码 | G.722 64kbps | LPDSP32 解码 |
| 音频输出 | OD (Sigma-Delta) | DIO8 (OD_P) / DIO9 (OD_N)，需 680μH 电感 |
| 输出采样率 | ~31.25kHz | AUDIOSLOWCLK 1MHz ÷ 64 |
| 音频时钟参考 | DIO7 (SAMPL_CLK) | 供 ASRC 补偿时钟漂移 |
| 按键 | DIO2 → DIO0 中断 | 短按切换 RM 开关 |
| 时钟要求 | **最低 16MHz** | 8MHz 会导致 CRC 错误/断连 |

## 十、低功耗进入条件（2026-07-02 验证）

### 核心三要素（缺一不可）

| # | 配置项 | 要求 | 位置 |
|---|--------|------|------|
| 1 | `RFCLK_FREQ` | **8MHz** | `app.h` |
| 2 | `BBIF->CTRL` | **`BB_DEEP_SLEEP`** | `app_init.c` |
| 3 | `app_env.audio_streaming` | **0** | `app_init.c` / `rm_app.c` |

### RM 对低功耗的影响

| RM 状态 | 能否进低功耗 |
|---------|:----------:|
| RM 完全关闭（`APP_RM_ENABLE` 注释） | 可以 |
| RM 初始化但不 enable（`APP_RM_Init` + `RF_SwitchToBLEMode`，不调 `RM_Enable`） | 可以 |
| RM 全开（`RM_Enable(1000)` + `audio_streaming=1`） | 不能 |

**结论：RM 代码编译进去、参数初始化都没问题，只要不 enable 且 audio_streaming=0 就能进低功耗。**

### 验证过程

1. `App_sleep_Initialize`（8MHz + `BB_DEEP_SLEEP` + 宏分频 + RM 关）→ 能进
2. 同配置改 16MHz → 不能进
3. `App_Initialize` 硬编码 16MHz + `BB_WAKEUP` + RM 开 → 不能进
4. `App_Initialize` 改宏分频 + `BB_DEEP_SLEEP` + 8MHz + RM 关 → 能进
5. 开 `APP_RM_ENABLE` 但注释掉 `APP_RM_Init` → 能进
6. 开 `APP_RM_Init` + `RF_SwitchToBLEMode`，不调 `RM_Enable` → 能进

### RM 与低功耗互斥

| | 低功耗模式 | RM 工作模式 |
|--|----------|-----------|
| 频率 | 8MHz | 16MHz |
| BBIF | `BB_DEEP_SLEEP` | `BB_WAKEUP` |
| CPCLK | 无 | `CPCLK_PRESCALE_8` |
| audio_streaming | 0 | 1 |

RM 和低功耗无法同时满足，需要**动态切换**：RM 空闲时切 8MHz + BB_DEEP_SLEEP 进低功耗，RM 工作时切 16MHz + BB_WAKEUP + CPCLK。
